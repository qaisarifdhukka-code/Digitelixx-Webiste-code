# 3. System Architecture
**Frontend Architecture**: Component-based React application using Next.js App Router. Pages are assembled in `src/app/page.js` by importing highly modular functional components from `src/components/`.
**Backend/Database/AI/Auth**: N/A. The site is a purely static front-end export.

**Architecture Diagram**:
```text
[Client Browser]
       |
       v
[Hostinger Shared Hosting]
       | (Serves Static HTML/CSS/JS)
       v
[out/ directory] <--- (Generated by `next build`)
       |
    [Next.js App Router Source]
       ├── /src/app/page.js (Main Layout Assembly)
       ├── /src/app/globals.css (Design System & Tokens)
       └── /src/components/ (Isolated UI blocks)
             ├── Hero.jsx
             ├── Services.jsx
             ├── CaseStudyHighlight.jsx
             └── Contact.jsx
```

# 17. Coding Standards
- **Naming conventions**: PascalCase for React components (`Hero.jsx`). camelCase for functions and variables. Kebab-case for CSS class names (`.case-study-split-grid`).
- **File organization**: One component per file in `src/components/`. Global styles isolated to `globals.css`.
- **Design patterns used**: Component Composition. Sections are isolated into their own files and imported into a central layout file to prevent monolithic files.
- **Error handling strategy**: Minimal error handling required for static rendering. Missing images should fail gracefully.
